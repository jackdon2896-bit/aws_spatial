🚀 DEPLOYMENT COMMANDS
================================================================================

OPTION 1: AWS HEALTHOMICS
------------------------
1. Upload to S3:
   aws s3 cp . s3://your-bucket/workflows/spatial/ --recursive

2. Create/Update workflow in HealthOmics Console

3. Run workflow:
   aws omics start-run \
     --workflow-id <id> \
     --name "spatial-test" \
     --parameters '{"tiff":"s3://bucket/image.tif","h5":"s3://bucket/data.h5"}' \
     --output-uri s3://bucket/output/ \
     --storage-capacity 1200 \
     --role-arn arn:aws:iam::account:role/HealthOmicsRole


OPTION 2: NEXTFLOW WITH AWS BATCH
----------------------------------
nextflow run main.nf \
  --tiff s3://bucket/image.tif \
  --h5 s3://bucket/matrix.h5 \
  --outdir s3://bucket/results/ \
  -profile awsbatch \
  -work-dir s3://bucket/work/


OPTION 3: LOCAL TESTING
-----------------------
nextflow run main.nf \
  --tiff test_data/image.tif \
  --h5 test_data/matrix.h5 \
  --outdir results/ \
  -profile docker


OPTION 4: SRA DATA SOURCE
--------------------------
nextflow run main.nf \
  --tiff s3://bucket/image.tif \
  --sra_ids "SRR15440796,SRR15440797" \
  --outdir s3://bucket/results/

⚠️ IMPORTANT NOTES
================================================================================

1. PYTHON SCRIPTS ARE STUBS
   The bin/*.py scripts create placeholder outputs. For production, you need
   to implement actual analysis logic using scanpy, squidpy, etc.

2. CONTAINER IMAGE
   Pipeline uses: public.ecr.aws/b2q5t5z5/spatial-omics:latest
   Rebuild if you need additional dependencies.

3. RESOURCE ALLOCATION
   Default: 4 CPUs, 16GB RAM per process
   Increase in nextflow.config for large datasets (>100k cells)

4. AWS PERMISSIONS
   IAM role needs: S3 read/write, ECR pull, HealthOmics execute

5. NEXTFLOW DSL2
   This pipeline uses modern Nextflow DSL2 syntax. It will NOT work with
   Nextflow <21.04.0. Tested with Nextflow 25.04.7.
